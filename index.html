Honeypot generator

A project by Bor de Kock, Pieter Kokx and Ylona Meeuwenberg

What is a honeypot?

Honeypot is a computer system that is setup as a decoy to lure attackers to gain unauthorized access.

Examples

  • A USB stick with 'secret' documents
    • With wrong information to catch traitors
  • Shared folder with secret documents

Honeypot detection

  • Check file metadata
  • Is the metadata consistent with normal file usage?

Literature

We were not able to find work on believable file usage.

There is literature on detecting honeypots

  • http://old.honeynet.org/papers/individual/DefeatingHPs-IAW05.pdf
  • http://ro.ecu.edu.au/cgi/viewcontent.cgi?article=1027&context=adf
However, all of these methods need to be able to execute programs on the machine. They don't look at file usage at all.

Almost no literature. So, we have to come up with our own approach here.

Approach

  • Analyse existing file metadata and generate a definition of believable file usage from there.
  • Apply this knowledge to manipulate files.

Why?

The main goal is to develop a methodology to analyse file data, to be able to generate file metadata in the future.

Variables to analyze

  • Filename (full path)
  • MIME-type (determined with magic)
  • Filesize
  • Last modified date (looking at commit history)
